Guidelines for the CGS Computer Officer

Responsibilities

The CGS computer officer is responsible for maintaining all the technical aspects of the society with respect to mailing lists, websites and printing. Specifically, these responsibilites include maintaining the following:

  1. Mailing list

  2. Printing term cards

  3. CGS website

  4. ISW website

  5. ISW Schedule

This document gives guidelines on each of these responsibilities.

Managing the Mailing list

The CGS mailing list: 

soc-cgs@lists.cam.ac.uk
is hosted by hermes.cam.ac.uk, the university mail server. A userid and password is required for hermes, which can be obtained from the Computing Services. Once logged into hermes, the mailing list can be managed by following the instructions for managing group email.

Printing Term Cards

One of the primary ways of advertising CGS events is through the distribution of term cards. These cards contain key information about the society as well as the events for the current university term. Information regarding the events is formulated by the CGS committee, and once this is available the new term card can be produced.

The term card file is in MS Word format so that it can be easily distributed, read and modified by other members of committee. A copy of the card is located in the directory storing the CGS web site. The term cards can be printed at St. Andrews the Great Church, which has generously offered to supply the card need for printing the term cards. Their phone number is 01223 518218.

The easiest way of printing the term cards is to lay each A4 side of the term card side-by-side [B1T1][T2B2] (where T refers to the top and B refers to the bottom of the landscape-oriented page) which together forms an A3 sheet. This is then copied twice with 70% reduction using 1->2 sided copying onto both sides of an A4 sheet of card. The sheet can then be cut into two A5 sections, and folded to form a mini-booklet.

Maintaining the CGS Website

The CGS website is fairly simple, consisting a small collection of static HTML pages. Mostly this involves updating the webpages of events, but once a year the page giving contact details for committee members needs to be updated. The website is hosted using the Group Filespace service at Computing Services, which has instructions for using this group filespace.

In particular, the files are located on the shared.pwf.cam.ac.uk FTP server. A userid and password needs to be obtained for this server and perrmission given to read and write files to the CGS directory. The following FTP commands are the most commonly used: 

help
cd SOC/cgs/WWW
ls
put file.htm
get file.htm

Maintaining the ISW Website

The ISW Website is located as a subdirectory of the CGS directory on shared.pwf.cam.ac.uk. It consists of a number of static HTML pages, and like the CGS website it is principally the events page that needs to updated each year. The only major difference is that the ISW website contains a form which can be used to contact members of the ISW committee. The email addresses for two of the hidden inputs for this form need to be updated each year: 

name="toText" value="email1@cam.ac.uk"
name="ccText" value="email2@cam.ac.uk"
where these two email addresses are for the people who will receive messages posted from the ISW website. The form is submitted to a mailtool utility located at http://www.srcf.ucam.org/cucgs/isw/mailtool/mailtool.php. This is a standard PHP mailtool that should not need to be changed.

Maintaining the ISW Schedule

One of the major contributions of ISW is to organize a large number of volunteers to meet new international students at the Cambridge train and coach stations. This requires keeping a schedule of the time commitments of the volunteers, and for a couple of years this has been stored using an on-line database. A MySQL database has been created, together with the appropriate PHP interface, and is hosted by SRCF. The computer officer needs to register with SRCF to obtain a username and a password, and must also be given permission to read and write files to the CGS directory.

Users of the ISW Schedule fall into two categories: superusers which can modify the contains of the database, and users who can only have minimal access to the database. The schedule itself can be viewed by anyone at the following URL: http://www.srcf.ucam.org/cucgs/isw/schedule/restricted/view/limited.php. Accounts can be created using http://www.srcf.ucam.org/cucgs/isw/schedule/restricted/lib/create-account.php?uid=youruserid. NOTE that the person creating these accounts must be logged into the database to created this new accounts. Also, usernames for superusers need to be added to the list in the is_superuser function in the file ~/cucgs/public_html/isw/schedule/restricted/include/connect-mysql.php.

The login page for superusers is located in the isw/schedule/ subdirectory of the CGS filespace on shared.pwf.cam.ac.uk. The database allows a person 15 minutes of inactivity before requiring them to re-login.

On the whole, the database should not need to be modified much from year-to-year. The contents of the database can be cleared by superusers using the PHP pages within the schedule. The biggest changes that need to made are the dates for ISW, which need to be changed manually in the following files: 

restricted/add/arrival-form.php
restricted/edit/arrival.php
restricted/edit/own.timeslot.php
restricted/edit/user.timeslot.php
restricted/include/decode.php
restricted/sign-up/sign-up.php
restricted/view/limited/coach.schedule.php
restricted/view/limited/index.html
restricted/view/limited/limited.php
restricted/view/limited/own.schedule.php
restricted/view/limited/train.schedule.php
restricted/view/limited/view.php
restricted/view/limited/whole.schedule.php
restricted/view/coach.schedule.php
restricted/view/limited.php
restricted/view/own.schedule.php
restricted/view/train.schedule.php
restricted/view/view.php
restricted/view/whole.schedule.php
To make the database more user-friendly, a PHP page for automatically updating these dates would be useful.

Questions?

For any questions on the responsibilities of the CGS Computer Officer, please contact the president of CGS, the Computing Services or SRCF.
